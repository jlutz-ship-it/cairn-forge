1

Doctrine

Operating Sequence
How this work is meant to be read
  1. Start with doctrine: read the BOP Protocol to understand the governance philosophy.
  2. Then inspect chronology: review the proof map to see when key artifacts appeared.
  3. Then inspect architecture: review CAIRN-K, Hotport, and related materials as technical design.
  4. Then inspect outreach: read the organization timelines and company-specific briefing drafts.
  5. Then inspect the annex: separate hard claims from working claims still under independent validation.
Validation note: Some claims in the wider Miskulin Technologies packet remain under independent validation. Those claims are intentionally isolated in a separate annex rather than mixed into the lead documents.
